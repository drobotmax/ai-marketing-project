# О Conversions API
![](https://static.xx.fbcdn.net/mci_ab/public/cms/?ab_b=e&ab_page=CMS&ab_entry=438223955063449&version=1666716831&transcode_extension=webp)
Вы можете получить пошаговые инструкции по внедрению Conversions API от технического специалиста по решениям Meta. Узнайте, [_как запланировать звонок со специалистом Meta_](https://www.facebook.com/business/meta-pro-team/technical-pro?lead_source=sbg_2025_q4_ou_specialist_technical_helpcenter_xyz).
Conversions API обеспечивает прямую связь между вашими маркетинговыми данными и системами Meta, которые помогают оптимизировать таргетинг рекламы, снижать цену за результат и измерять показатели на платформах Meta.
## Принцип работы Conversions API
Conversions API обеспечивает прямую и надежную связь между Meta и маркетинговыми данными, например [событиями на сайте](https://www.facebook.com/business/help/964258670337005), [в приложении](https://www.facebook.com/business/help/235457266642587), [офлайн-конверсиями](https://www.facebook.com/business/help/1142103235885551) и [событиями переписки](https://www.facebook.com/business/help/610611921269482) с вашего сервера, сайта, приложения или из CRM-системы. Эти данные помогают персонализировать и оптимизировать рекламу, а также измерять ее результаты на платформах Meta. Так вы сможете показывать объявления [людям](https://www.facebook.com/business/help/283579896000936), которые с большей вероятностью ими заинтересуются.
Офлайн-конверсии можно использовать для оптимизации рекламы в кампаниях с целью "Продажи", если вы выберете место получения конверсий "Сайт и магазин".
Существует несколько способов настроить Conversions API. Многие интеграции можно выполнить без помощи разработчика. Подробнее о том, [как выбрать оптимальный для вашей компании способ настройки](https://www.facebook.com/business/help/433493041367251).
## Управление данными о событиях с помощью Conversions API
Conversions API позволяет управлять данными о событиях с помощью единого API. Conversions API поддерживает события из различных источников. Это могут быть сайты, физические точки, электронная почта, бизнес-чаты, телефон, мобильное приложение, а также офлайн-ресурсы. Вместо App Events API и Offline Conversions API вы можете настроить Conversions API. Если вы уже настроили App Events API или Offline Conversions API и не хотите создавать наборы данных, то можете продолжать использовать эти инструменты.
**Примечание.** Поддержка Offline Conversions API прекратится в мае 2025 года. Вы не сможете использовать Offline Conversions API, чтобы загружать события в группы офлайн-событий. Рекомендуем загрузить события в группы событий и интегрировать Conversions API вместо Offline Conversions API. [Подробнее об этих изменениях и подготовке к ним.](https://www.facebook.com/business/help/1835755323554976)

### Отправка событий на сайте с помощью Conversions API
Отправляя события на сайте с помощью Conversions API, вы сможете:
  * **Снизить цену за результат благодаря улучшению связи.** Ошибки при загрузке браузера, проблемы с подключением и блокировщики рекламы меньше влияют на данные Conversions API, чем на данные пикселя Meta. Использование Conversions API в сочетании с пикселем обеспечивает более надежную связь и помогает системе показа рекламы снизить цену за результат.
  * **Оптимизировать рекламу для увеличения числа действий на всем дальнейшем пути клиента.** К этим событиям относятся действия после покупки (например, подписки), действия в магазине и оценки клиентов. Эта информация помогает системе показывать вашу рекламу клиентам, которые с большей вероятностью совершат ценное для вашей компании действие.
  * **Повысить точность измерения.** Conversions API помогает точнее измерять результативность и атрибуцию рекламы на всех этапах пути клиента: от знакомства с брендом до конверсии. Вы сможете лучше понимать, как цифровая реклама влияет на онлайн- и офлайн-результаты вашей компании.
  * **Снизить цену за результат благодаря улучшенному сопоставлению событий.** Совпадающие события — это события, которые можно атрибутировать аккаунтам на платформах Meta. Сопоставление событий помогает показывать рекламу людям, которые с большей вероятностью выполнят нужное вам действие, а затем атрибутировать эти действия вашей рекламе. Благодаря Conversions API вы сможете добавлять дополнительные параметры информации о клиентах, чтобы повышать количество совпадающих событий и качество их сопоставления. Подробнее о том, как [просматривать](https://www.facebook.com/business/help/1541268312717919) и [улучшать](https://www.facebook.com/business/help/308855623839366) качество сопоставления событий.


## Интеграция Conversions API с пикселем Meta
Если вы используете Conversions API для отправки событий на сайте, сочетайте этот инструмент с пикселем, чтобы получать максимально эффективные результаты. [Подробнее о рекомендациях по Conversions API.](https://www.facebook.com/business/help/308855623839366)
События на сайте, полученные через Conversions API, связаны с вашим пикселем и функционируют как события пикселя в следующих случаях:
  * Они используются для тех же типов оптимизации рекламы, в том числе максимального числа конверсий и максимальной ценности конверсий.
  * Они появляются на большинстве тех же платформ, в том числе в Meta Ads Manager и Meta Events Manager.
  * На них распространяются ограничения, определенные в [Условиях использования инструментов Meta для бизнеса](https://www.facebook.com/legal/technology_terms). Событиями на сайте можно управлять с помощью инструмента "Действия вне Facebook" и сторонних решений для контроля данных при персонализации рекламы.
  * Как и пиксель, Conversions API не предназначен для обхода правил в отношении передачи данных (например, фреймворка App Tracking Transparency на устройствах iOS) и действующих в Европе норм конфиденциальности, таких как Директива о конфиденциальности и электронных средствах связи.


Подробнее о том, [как отправлять события на сайте через прямую интеграцию Conversions API](https://developers.facebook.com/docs/marketing-api/conversions-api/get-started) (сайт Meta for Developers).

### Отправка событий в приложении с помощью Conversions API
Отправляя события в приложении с помощью Conversions API, вы сможете:
  * **Меньше зависеть от SDK и релизов обновлений приложения.** Оправляйте новые данные о событиях с помощью Conversions API. При этом вам не нужно использовать Facebook SDK для iOS или Android и зависеть от релизов обновлений приложения.

### Отправка событий переписки на сайте с помощью Conversions API
Отправляя события переписки с помощью Conversions API, вы сможете:
  * **Подключать ценные данные из бизнес-чата.** Отправка событий поможет понять, какие действия люди выполняют в бизнес-чатах в Messenger, Instagram или WhatsApp.
  * **Оптимизировать рекламу с переходом в Messenger для покупок, чтобы повышать результативность.** Сейчас оптимизация доступна только для событий покупки в Messenger. [Подробнее о событиях переписки.](https://www.facebook.com/business/help/610611921269482)
  * **Повышать точность измерения.** Conversions API поможет лучше понимать и анализировать действия, происходящие в конце пути клиента (например, покупки).

## Будьте в курсе новостей от Meta for Business
