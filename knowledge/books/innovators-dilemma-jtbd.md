# Клейтон Кристенсен — Дилемма инноватора + Jobs to be Done

## Часть 1: Дилемма инноватора

### Суть теории
Великие компании терпят поражение именно потому, что делают всё правильно — слушают клиентов, инвестируют в лучшие технологии, максимизируют прибыль. Это делает их уязвимыми к разрушительным инновациям.

### Два типа инноваций

**Поддерживающие (Sustaining):** Улучшают продукты по параметрам, которые уже ценит рынок. Доминирующие компании почти всегда выигрывают.

**Разрушительные (Disruptive):** Первоначально хуже по ключевым параметрам. Дешевле, проще, удобнее. Два вектора:
- **New-market disruption** — создание рынка для не-потребителей
- **Low-end disruption** — атака снизу через непривлекательный для лидера сегмент

### Пять механизмов провала лидеров

1. **Зависимость от клиентов** — клиенты не просят о разрушительных инновациях
2. **Проблема размера** — новые рынки слишком маленькие для крупных игроков
3. **Ловушка данных** — несуществующие рынки не поддаются анализу
4. **Performance Overshoot** — продукт улучшается быстрее чем нужно рынку
5. **RPV-ловушка** — Resources, Processes, Values оптимизированы под текущий бизнес

### RPV Framework

| Компонент | Что это | Почему ловушка |
|-----------|---------|---------------|
| **Resources** | Люди, технологии, деньги | Можно купить — самый гибкий |
| **Processes** | Как принимают решения | Оптимизированы под текущий бизнес |
| **Values** | Приоритеты компании | Определяют что попадает в бюджет |

### Иерархия конкуренции (4 фазы рынка)
1. Функциональность → 2. Надёжность → 3. Удобство → 4. Цена

### Стратегии ответа
- Создание независимого подразделения с собственными RPV
- Планы для обучения, а не для исполнения
- Поиск нестандартных клиентов
- Создание новых рынков vs конкуренция на существующих

## Часть 2: Jobs to be Done (JTBD)

### Фундаментальная идея
"Люди не покупают продукты — они нанимают их для выполнения работы"

### Три измерения работы

| Тип | Описание | Пример |
|-----|----------|--------|
| **Functional** | Практическая задача | "Добраться на работу за 30 мин" |
| **Emotional** | Желаемое ощущение | "Чувствовать себя продуктивным" |
| **Social** | Как выглядеть в глазах других | "Выглядеть профессиональным" |

Эмоциональные и социальные работы в 2–3 раза сильнее влияют на решение.

### Канонический пример: молочный коктейль McDonald's
- **Утро:** водитель едет на работу — работа "сделать скучную дорогу терпимой"
- **День:** родитель с ребёнком — работа "дать маленькое удовольствие"
- Один продукт — две принципиально разных работы

### Четыре силы прогресса (Four Forces)

**Силы изменения:**
- **Push** — разочарование текущим решением
- **Pull** — привлекательность нового

**Силы сопротивления:**
- **Habit** — привычка к текущему
- **Anxiety** — тревога о новом

**Покупка происходит:** Push + Pull > Habit + Anxiety

### Применение в маркетинге по силам
- **TOFU:** адресует Push и Pull
- **MOFU:** снижает Anxiety (кейсы, гарантии)
- **BOFU:** преодолевает Habit (онбординг, миграция)

### Switch Interview (10 шагов, ~47 мин)
1. Введение
2. Мир до покупки
3. Первая мысль о смене
4. Момент фрустрации
5. Поиск альтернатив
6. Почему именно это решение
7. Первый опыт
8. Эволюция использования
9. Текущие боли
10. Анализ четырёх сил

### Outcome-Driven Innovation (ODI) — метод Улвика

**Job Map (8 шагов):** Define → Locate → Prepare → Confirm → Execute → Monitor → Modify → Conclude

**Opportunity Score:** `Importance + max(Importance - Satisfaction, 0)`

### Позиционирование через JTBD
```
[Продукт] помогает [аудитория по контексту]
когда [ситуация/триггер]
[выполнить работу]
без [главная тревога/барьер]
```

### JTBD vs Персоны
- Персоны описывают **кто** покупает — JTBD объясняет **почему**
- Демография не предсказывает поведение
- Сегменты строятся по схожести работы, не демографии

### JTBD для B2B
Работу выполняет конкретный человек, но на решение влияют stakeholders:
- Менеджер: "Не облажаться с выбором вендора"
- Пользователь: "Выполнить задачу быстро"
- CTO: "Не создать технический долг"
- CFO: "Обосновать ROI"

Каждому — отдельный messaging.

## Ключевые инсайты

1. Контекст важнее демографии
2. Настоящие конкуренты определяются работой, не категорией
3. Покупка — это найм. "Увольняют" когда Push + Pull > Habit + Anxiety
4. Работы не меняются, продукты — меняются
5. Первые 2 года в disruptive рынке = в 20 раз больше выручки
6. Лучший маркетинговый текст пишут сами клиенты (switch-интервью)
