# Case: Allex Villas (Bali real estate developer)

Второй reference-кейс методологии web-структуры — для **B2C-сайта застройщика** (девелопер премиальной недвижимости на Бали).

Дополняет [case-uph](../case-uph/) (B2B-каталог производителя) недвижимостным треком: инвест-вилла-комплекс, multi-language, мудборд → концепты, SEO под YMYL-нишу.

## Контекст проекта

- **Клиент:** Alex Villas Group (alexvillasgroup.com), full-cycle Bali property developer с 2014 г.
- **Задача:** редизайн и обновление действующего сайта.
- **Команда (KUBRIK):** проектировщик, ОС-дизайнер (концепты), SEO-специалист, разработчик.
- **Период:** 2025-08 → 2025-11 (паспорт утверждён 25.08.2025, структура — 16.09.2025, концепты — 25.11.2025).
- **Статус на момент передачи:** дофинала не доехало, передача в работу 30.04.2026.

## Чем этот кейс отличается от UPH

| Аспект | UPH (B2B) | Allex Villas (B2C real estate) |
|---|---|---|
| Тип сайта | Каталог-сайт без покупки | Девелоперский лендинг + листинг проектов |
| ЦА | B2B (отрасли + госсектор) | B2C-инвестор + multi-language |
| Каталог | Технология / отрасль / вид | Услуги + Разработки + Продажа + Аренда + сегментация по 5 фильтрам |
| SEO-режим | Корп-сайт + блог-новости | YMYL (Your Money Your Life): жёсткие требования к E-E-A-T |
| Дизайн-фаза | Wireframe → ТЗ → дизайн | Wireframe → **мудборд** → 2-3 итерации концептов → ТЗ |
| Артефакт KUBRIK | Прототип Excalidraw + ТЗ | Прототип Excalidraw + Figma-концепты + 4 SEO-документа |
| Допы между этапами | - | СЕО-анализ КФ + анализ конкурентов + семантика + гибридная структура + мета-теги (отдельный SEO-этап после структуры) |

## Ключевые методологические инсайты кейса

1. **Гибридная структура = SEO-структура × бизнес-структура.** Для девелоперской недвижимости каталог объектов разворачивается через **5 параллельных осей фильтрации**: тип недвижимости, тип собственности, бюджет, локация, удобства/спален. Это создаёт SEO-страницы под кластеры запросов («villa for sale Canggu», «freehold bali», «1 bedroom villa for rent» и т.д.).
2. **YMYL-режим обязывает к E-E-A-T-блокам.** Недвижимость = деньги, поэтому Google требует: блог с авторами (фото+ФИО+соц), schema.org/Article, отзывы, лицензии, портфолио, «о компании» с командой. На каждой странице блог-статьи — блок «об авторе».
3. **Мудборд между структурой и концептами.** В отличие от B2B-флоу UPH, в недвижимости после структуры идёт **отдельный созвон по мудборду** (визуальный референс), затем 2 итерации концептов (06.11 → 20.11) и финальные «ОС-концепты» (25.11). Дизайнер ОС работает не сразу с прототипом, а через мудборд → концепт → правки.
4. **Анализ конкурентов = метрики + E-E-A-T-аудит.** SEO-команда строит таблицу по 9 параметрам (Age, DR, Organic Traffic, Organic KW, Referring Domains, Core Web Vitals mobile/desktop, AI Visibility, Structured Data, E-A-T-комментарий). Считается **медиана** по нише — целевой ориентир для нашего сайта.
5. **Анализ КФ (коммерческих факторов)** — отдельный артефакт между структурой и прототипом. Сравнивает блоки нашего сайта с тем, что есть у каждого второго конкурента. Каждый блок отмечается «реализуем / нет / под сомнением» в комментариях клиента (см. `seo-blocks-tz.md`).
6. **Multi-language как структурное решение.** Для Бали-проекта основной язык EN (международный инвестор), но переключатель и мета-теги предусматриваются с первого дня.
7. **Калькулятор рентабельности + Прогноз доходности** — обязательные коммерческие блоки на странице проекта и в основной навигации. Для инвест-недвижимости это центральный CTA.
8. **Каннибализация листингов.** Если у застройщика несколько листингов одного типа («Strоится / Построено / Готовые / Со скидкой / Виллы высокого класса» и т.д.) — каждый листинг должен иметь **уникальный текстовый блок** (FAQ, описание категории), иначе Google склеит и не будет ранжировать.
9. **Footer ≠ юр. инфо.** В нём — меню коммерческих и информационных страниц + ссылки на все соцсети, где идёт реклама проектов (для индексации связок).
