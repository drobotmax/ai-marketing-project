# Гибридная структура сайта (Allex Villas, B2C девелопер)

Финальная sitemap, утверждённая 16.09.2025. «Гибридная» = бизнес-структура × SEO-структура: каталог объектов разворачивается через 5 параллельных осей фильтрации, каждая ось рождает посадочные страницы под кластеры запросов.

## Принципы гибридной структуры (для применения на других девелоперах)

### 1. Услуги ≠ Объекты
- **«Услуги»** = что компания умеет делать (строительство, управление, юр. сопровождение, …). Лендинговые описания. SEO под кластеры «<услуга> bali».
- **«Разработки» / «Продажа» / «Аренда»** = объекты как товар. Листинги + карточки. SEO под кластеры «villa for sale Canggu», «freehold bali» и т.д.

### 2. Каталог разворачивается через 5 осей
- **Тип объекта** (квартира / вилла / земля / отель)
- **Тип собственности** (аренда / фрихолд) — критично для иностранных инвесторов
- **Бюджет** (4 диапазона)
- **Локация** (5–N направлений)
- **Удобства** или **спальни** (для аренды)

Каждая ось = отдельная посадочная страница со своим title/description, отфильтрованным листингом и уникальным текстом FAQ. **Без уникального текста — каннибализация выдачи.**

### 3. Текущие проекты ≠ Продажа
- **«Разработки»** = что компания строит сейчас (3 типа: жилая / отели / коммерческая). Это страница про **компетенции девелопера**.
- **«Продажа»** = маркет-плейс, где можно купить объект (свой или сторонний).

Для девелоперов с собственным портфелем эти разделы могут пересекаться по карточкам, но листинги разные: «Разработки» сортируется по статусу строительства, «Продажа» — по запросам инвестора.

### 4. Аренда — вторая воронка
Для девелопера аренда — способ удерживать клиента после продажи (управление + ROI) и привлекать инвесторов через демонстрацию доходности. Аренда **обязана** быть отдельным разделом, иначе страдает SEO под «bali villa for rent» (147 запросов в кластере).

### 5. Инвест-инструменты в основной навигации
- **Калькулятор рентабельности** — отдельный пункт меню, не футер. Это центральный CTA для инвестора.
- **Прогноз доходности** — обязательный блок на странице каждого проекта.
- **Диверсифицировать портфель** + **Инвестиции на Бали** — отдельные посадочные с инвест-логикой.

### 6. E-E-A-T-блоки в навигации
- **Наша команда** + **Карьера** под «О компании» — для YMYL-ниши Google требует «человеческое лицо».
- **FAQ** в верхнем меню — sitewide-страница вопросов, не только FAQ-блоки на страницах.
- **Блог / Новости** — обязательно с авторами статей и schema.org/Article.

### 7. Партнёрское плечо
- **Корпоративное партнёрство** — отдельная страница для агентств, юристов, архитекторов, кто хочет приводить клиентов. Аналог B2B-страницы «Дилерам» в UPH-кейсе.

## Как использовать структуру на новом проекте

1. Снять с клиента: какие из 7 услуг есть, какие типы объектов, какие локации, в каких валютных диапазонах работает.
2. Снять параллельно с SEO-командой кластеры запросов.
3. Свернуть оси, под которые **нет ни запросов, ни объектов** (например, у клиента нет коммерческой недвижимости — выкидываем ось).
4. Развернуть оси, под которые **есть запросы, но нет страниц** — это рост-возможности.
5. Структуру утверждать с клиентом + SEO-специалистом одновременно: бизнес-логика + поисковый спрос.
