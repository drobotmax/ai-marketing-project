# Web Structure Methodology

База знаний по эталонному процессу разработки сайта «с нуля»: паспорт проекта → структура → (SEO-этап → мудборд) → прототип → ТЗ → концепты → разработка.

## Два reference-кейса

- **[case-uph](case-uph/)** — B2B-каталог-сайт производителя (UPH, аналитическое/досмотровое оборудование). Базовый флоу. Каталог + госзакупки + дилеры + сотрудничество.
- **[case-allex-villas](case-allex-villas/)** — B2C-сайт девелопера недвижимости (Allex Villas, Бали). Расширенный флоу с SEO-этапом, мудбордом, итерациями концептов. Гибридная структура с 5 осями фильтрации, YMYL-режим (E-E-A-T).

Используйте UPH-кейс для B2B-каталогов, Allex Villas — для B2C-недвижимости и других YMYL-ниш.

## Когда обращаться к этой базе

Claude должен читать и применять эту методологию в случаях:
- Клиент просит **разработку сайта/лендинга/корп. сайта/каталога** с нуля.
- Нужно спроектировать **структуру сайта** (B2B-каталог или B2C-листинг недвижимости).
- Нужен **чеклист блоков** для главной / листинга / карточки / отраслевой / блог-страницы.
- Запрос на **паспорт проекта** (бриф с целями, ЦА, CJM).
- Клиент с **B2B / госсектором / производством** — UPH-режим (госзакупки, дилеры, КБ, вузы).
- Клиент **застройщик / девелопер / агентство недвижимости** — Allex Villas-режим (гибридная структура, YMYL E-E-A-T, мудборд → концепты, инвест-калькулятор).
- Нужно понять, **какие вопросы задавать клиенту** на созвоне по структуре (фиксация «пустот» на доработку).
- Нужен **SEO-этап** между структурой и прототипом (анализ конкурентов, семантика, КФ, гибридная структура, мета-теги).
- Ценообразование на **прототип + ТЗ + SEO + концепты** для референса.

## Какой режим выбрать

| Признак | UPH (B2B-каталог) | Allex Villas (B2C-RE) |
|---|---|---|
| Что продают | оборудование / ПО / услуги бизнесу | объекты недвижимости / инвест-возможности |
| Конечный покупатель | B2B-закупщик / отдел закупок | конечный инвестор / семья |
| YMYL | нет | **да** (Google требует строгий E-E-A-T) |
| Каталог | 3 группировки (технология/отрасль/вид) | гибридный, 5 осей фильтрации |
| Доп. треки | госзакупки, дилеры, сотрудничество | управляющая компания, инвест-инструменты |
| Доп. блоки | сервис после гарантии, схема предприятия | калькулятор ROI, прогноз доходности |
| Multi-language | редко | часто (международный инвестор) |
| Дизайн-флоу | 1 итерация после прототипа | мудборд → 2-3 итерации концептов |
| SEO-этап | блог + базовая семантика | **отдельный этап:** анализ конкурентов + 9-параметровая таблица + ТЗ КФ + гибридная sitemap |

## Ключевые инсайты UPH (B2B)

1. **3 варианта группировки каталога** (технология / отрасль / вид) — выбор зависит от ЦРМ/1С клиента; без сверки с ERP каталог придётся переделывать.
2. **Бургер-меню каталога** как современный SEO-паттерн для большой номенклатуры.
3. **Пустая поисковая выдача → форма обратной связи**, а не «ничего не найдено». Не отпускаем заинтересованного пользователя.
4. **Отраслевые страницы с интерактивной схемой предприятия** (входная группа / лаборатория) + кликабельные карточки оборудования в схеме.
5. **Госзакупки = отдельная страница** (B2G-трек), если клиент работает с 44-ФЗ/223-ФЗ.
6. **Дилеры = отдельная страница + отдельный процесс**: спросить у клиента, как работает входящий лид дилера (анкета vs форма), нужно ли показывать существующих дилеров на карте (часто нет — «рынок захвачен» демотивирует новых).
7. **Сотрудничество разделяется**: научно-конструкторские бюро + вузы. У каждого своя логика, форма, отзывы.
8. **Сервис после гарантии** = отдельный блок удержания клиентов (не теряем их после окончания гарантийного периода).
9. **Формы обратной связи**: выпадашка «отрасль» нужна только если у клиента в ОП менеджеры разделены по отраслям.
10. **Price на сайте** — только если в компании есть кто-то, кто следит за актуальностью. Иначе лучше не размещать.

## Ключевые инсайты Allex Villas (B2C-недвижимость)

1. **Гибридная структура = SEO × бизнес.** Каталог объектов разворачивается через 5 параллельных осей: тип / собственность / бюджет / локация / спальни.
2. **YMYL обязывает к E-E-A-T-блокам.** Блог с авторами, schema.org/Article, отзывы с ФИО, лицензии, портфолио, команда с фото.
3. **Мудборд между структурой и концептами.** Без мудборда — 5 итераций «не той эмоции», с ним — 2 итерации до финала.
4. **SEO-этап до прототипа.** Анализ конкурентов (9 параметров) + семантика + ТЗ КФ + гибридная sitemap + мета-теги. Делается параллельно структуре, до прототипа.
5. **Анализ КФ как чеклист.** Каждый блок — «есть у каждого второго конкурента / нет / под сомнением» с разметкой клиента.
6. **Multi-language structural.** EN основной для международного инвестора; RU/ZH/FR опционально. Hreflang обязателен.
7. **Калькулятор рентабельности + Прогноз доходности** — обязательные коммерческие блоки в навигации и на странице проекта.
8. **Каннибализация листингов.** Несколько листингов одного типа → каждый со своим уникальным FAQ.
9. **Footer ≠ юр-инфо.** В нём — меню коммерческих + информационных страниц + ссылки на все соцсети с рекламой проектов.
10. **Команда — обязательный E-E-A-T-блок.** Без «человеческого лица» Google не доверяет YMYL-сайту.

## Паттерн работы проектировщика

### B2B-флоу (UPH)
1. **Паспорт** (созвон 1) — цели проекта, ЦА, позиционирование, конкуренты.
2. **Структура** — Miro со структурой + варианты для спорных моментов.
3. **Внутренняя проверка** — созвон с коллегой (мини-walkthrough).
4. **Демо клиенту** — пошаговый разбор; фиксируются «пустоты».
5. **Финализация структуры**.
6. **Прототип** (Excalidraw / Figma wireframes).
7. **ТЗ**.
8. **Дизайн + разработка**.

### B2C-RE-флоу (Allex Villas)
1. **Паспорт** — + бизнес-модель девелопера, инвест-инструменты, юр-структура.
2. **Структура** — гибридная sitemap.
3. **SEO-этап** (новый) — анализ конкурентов + семантика + ТЗ КФ + гибридная sitemap + мета-теги.
4. **Прототип** — wireframes + калькулятор доходности.
5. **Мудборд** (новый) — созвон с дизайнером + клиентом, согласование «эмоции» бренда.
6. **Концепты** — 2-3 итерации (главная → ключевые страницы → финал).
7. **ТЗ** — + schema.org + multi-language + hreflang.
8. **Разработка** — + калькуляторы + фильтры + индексация листингов.
