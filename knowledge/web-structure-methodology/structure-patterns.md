# Паттерны структуры B2B-сайта (каталог-тип)

Чеклист блоков и решений для проектирования B2B-сайта производителя/дистрибьютора с каталогом без онлайн-покупки.

## Шапка (верхнее меню)

- Логотип
- Слоган (1 строка, чем мы занимаемся)
- Контакты (телефон, email)
- Price - только если в компании кто-то следит за актуальностью. Открывается в отдельном окне / скачивается.
- Языковые версии (если есть экспорт)
- Кнопка "получить консультацию" (открывает попап с формой)
- Поиск

**Форма "получить консультацию":**
- Заголовок + короткий текст
- Поле "имя"
- Выпадашка "отрасль" - **только если ОП разделён по отраслям**
- Поле "телефон"
- Выбор удобного способа связи: звонок, Max, другие мессенджеры - по политике клиента
- Поле "email" (обязательность - спросить у клиента)
- Кнопка "отправить"
- Попап "спасибо, специалист свяжется"

## Меню каталога (бургер + выпадающее)

Большое выпадающее меню каталога с группировкой - современный SEO-паттерн.

**3 варианта группировки (выбор зависит от ЦРМ/1С и ЦА):**
1. По технологии оборудования (аналитическое / досмотровое / ...)
2. По отрасли применения (металлургия / медицина / таможня / ...)
3. По виду (оборудование / ПО / методические пособия / комплексы)

**Правило:** всегда обсудить с клиентом и сверить с 1С, чтобы избежать переделок.

## Основное меню (после шапки)

7-8 пунктов максимум, чтобы поместились в одну линию:
- Каталог (бургер)
- Готовые решения по отраслям
- Госзакупки (если работаете с 44-ФЗ/223-ФЗ)
- Дилерам
- Сотрудничество
- О компании
- Блог / Новости
- Контакты

## Главная страница

1. **Промоблок** - заголовок, описание компании, кнопка "узнать больше о нас"
2. **Компания в цифрах** - статистика (лет на рынке, клиентов, проектов)
3. **Преимущества работы с нами** - 3-6 пунктов с расшифровкой
4. **Полноценный каталог** - карточки категорий (картинка, название, список под-категорий, "подробнее")
5. **Кнопка "скачать каталог / прайс"** (опционально)
6. **Блок для дилеров** - короткий текст, кнопка "подробнее"
7. **Готовые решения по отраслям** - плитка с визуалом отрасли
8. **Форма обратной связи**
9. **Почему это важно** - миссия / законодательство / безопасность
10. **Клиенты** - логотипы + благодарственные письма со сканами (proof)
11. **Сертификаты**
12. **Портфолио** - объекты (фото здания, оборудование, отзыв представителя, если можно)
13. **Блок сотрудничества** (для КБ) - короткий текст + кнопка
14. **Новости** - слайдер карточек + "смотреть все"

**Принцип:** каждый блок главной = ссылка на страницу из меню. Главная = дайджест.

## Страница каталога

- Хлебные крошки
- Разделение на группы (аналитическое / досмотровое - или другие по ЦРМ)
- В каждой группе - карточки под-категорий (картинка, название, список, "подробнее")
- Переход в под-категорию

## Страница под-категории

- Хлебные крошки
- Промоблок: заголовок, краткое описание "какие задачи решает категория", картинка типового оборудования
- **Подборка "для каких объектов применяется"** - кликабельные иконки отраслей, ведут на страницы отраслей
- (Опционально) Упоминание федерального закона, если есть единый регулирующий документ
- Плитка карточек конкретного оборудования (компактные карточки, не друг под другом)
- **Блок "оборудование под ключ"** - обслуживание, установка, настройка, обучение
- **Блок "сервис и ремонт"** - гарантийное + послегарантийное (удержание клиента)
- Преимущества компании
- Портфолио объектов по этой категории
- Отзывы
- Форма обратной связи

## Детальная карточка товара/оборудования

- Хлебные крошки
- Слайдер фото + основная фотография
- Заголовок + **кнопка "получить консультацию"** (CTA сразу)
- Ёмкое назначение оборудования (2-3 предложения)
- (Опционально) Федеральный закон, регулирующий установку
- Области применения - **кликабельные отрасли**, ведут на страницы отраслей
- **Форма обратной связи** (середина страницы)
- Преимущества оборудования - **на картинке со стрелочками** (если применимо), или список
- **Схема "как мы работаем"** - пошагово: выбор -> оформление -> установка -> обучение -> поддержка -> пост-гарантия
- Скачать прайс по этому оборудованию
- Документы и лицензии (можно скачать/посмотреть)
- Технические характеристики (таблица)
- Форма обратной связи
- (Опционально) Новости по этому оборудованию (SEO)

## Страница отрасли (готовое решение)

- Промоблок с картинкой и анонсом
- Описание задач, которые решает группа оборудования в этой отрасли
- **Интерактивная схема предприятия** (входная группа / лаборатория) - при наведении на оборудование всплывает карточка, при клике - переход на страницу оборудования
- Список оборудования для отрасли (карточки)
- Форма обратной связи
- Установка под ключ, сервис
- Портфолио по отрасли
- Преимущества для **этой конкретной отрасли** (не общие)
- Новости по отрасли (SEO + информационная миссия)
- Форма обратной связи

## Страница госзакупок

Отдельная страница для B2G-клиентов (госучреждения).
- Преимущества работы с компанией именно по 44-ФЗ / 223-ФЗ
- Каталог (дубль)
- Форма обратной связи

## Страница дилерам

- Промоблок
- Презентация (если есть)
- Текст "как стать дилером"
- Информация о компании (лидеры в сфере)
- Преимущества сотрудничества (3-6 пунктов)
- О продукции (каталог)
- Форма обратной связи
- (**НЕ** показывать текущих дилеров на карте - демотивирует потенциальных)

## Страница сотрудничества

Разделяется на типы клиентов, у каждого своя страница:
- Научно-исследовательские и конструкторские предприятия (КБ)
- Вузы

**На каждой:**
- Промоблок
- О компании
- Какие партнёры интересны (характеристики - вид деятельности, опыт, документы) - чтобы фильтровать лиды
- Варианты сотрудничества + преимущества для партнёра
- С кем уже работаем (если можно - логотипы, совместные проекты, отзывы)
- Форма обратной связи
- Новости по тематике (наука и технологии)

## Страница о компании

- Картинка + описание
- Миссия и принципы
- Направление деятельности
- Наше оборудование (краткий обзор)
- Почему стоит работать с нами (особенности)
- Форма обратной связи
- Сертификаты
- Клиенты (логотипы)
- Портфолио
- Скачать каталог / прайс
- Новости компании

## Блог / Новости

- Промоблок
- Список статей (карточки с изображением, заголовком, анонсом)
- Детальная страница статьи (SEO-оптимизированная)
- Теги / категории (по отраслям, по типам оборудования)

## Контакты

- Офис (адрес, карта)
- Производство (если есть)
- Телефоны
- Email
- Мессенджеры
- Форма обратной связи

## Футер

- Логотип
- Контакты
- Поиск (дубль)
- Copyright
- Дубль главного меню
- Дубль каталога (под-категории)

## Специальные состояния

- **Пустой поиск** - форма обратной связи, а не "ничего не найдено". Предложить позвонить / написать / получить консультацию.
- **Отправка формы** - попап "спасибо, специалист свяжется"
- **404** - переход на главную / каталог

## Шапка (RE-режим)

Доп. к общему чеклисту:
- Переключатель языков (EN основной, RU/ZH/FR опционально)
- Кнопка «Calculate ROI» / «Рассчитать доходность» (если есть калькулятор)
- Кнопка «Book a viewing» / «Записаться на просмотр» (если работаете с physical-визитами)
- Контакты ассистента (WhatsApp / Telegram / Phone) видны сразу

## Главная страница (RE)

Стандартный лендинг застройщика. 15 ключевых блоков, проверенных против конкурентной ниши:

1. **Меню + быстрые контакты + поиск + язык**
2. **Видео/изо-баннер** + 2 кнопки (одна на основную услугу, одна на просмотр)
3. **Миссия компании** (1 экран, 2-3 предложения)
4. **Ценности** (3-5 пунктов)
5. **Мы в цифрах** (лет на рынке, проектов, клиентов, $ инвестиций под управлением)
6. **Превью услуг** — все услуги текстом, при наведении меняется иллюстрация (не слайдер!)
7. **Портфолио** — интерактивная карта объектов + превью + кнопка к полному портфолио
8. **Процесс работы** — таймлайн от первого контакта до владения
9. **Блог / Новости** — превью + переходы; разделить новости и информационный контент
10. **Авторитетные источники** — упоминания в СМИ, лицензии, награды, ассоциации (объединить с CTA «сотрудничать»)
11. **Награды и премии** (если есть)
12. **Отзывы** — слайдер с цитатами, **с ФИО + датой**, чтобы выглядело подлинно
13. **FAQ** (опционально, для текстовой релевантности)
14. **Форма обратной связи**
15. **Сквозной футер** — юр. + контакты + 2 столбца меню (коммерческое/информационное) + ссылки на все соцсети с рекламой проектов

## Страница списка проектов (листинг)

1. **Сквозной хедер**
2. **Баннер** — изображение + УТП + кнопка «связаться»
3. **Листинг карточек** с фильтром «Строится / Построено»
   - Карточка: название, фото, краткое описание, основные характеристики, кнопка «Узнать подробнее»
   - **Доп. в карточке:** пиктограмма «продаётся / продано», локация, минимальная цена болдом
4. **Блок FAQ** — **критичен**, если на сайте несколько листингов одного класса (отели/жилая/коммерческая) — иначе каннибализация выдачи
5. **Плитка тегов** (опц.) — анкорные ссылки внутри сайта с ключами
6. **Форма обратной связи**
7. **Сквозной футер**

## Страница проекта (детальная карточка)

1. **Сквозной хедер**
2. **Превью проекта** — фото + краткие характеристики + CTA «связаться»
3. **Навигационное меню по странице** (якорное оглавление)
4. **Описание проекта** — фото/визуализация + текст
5. **Локация** — карта + описание + преимущества + ссылки на другие проекты в этой локации (перелинковка)
6. **Прогноз доходности** — поднять выше в макете; сразу под — CTA «связаться»
7. **О комплексе** — архитектурный проект + инженерная информация
8. **Форма обратной связи**
9. **Дорожная карта проекта** (этапы строительства)
10. **Документация** (что можно скачать)
11. **Команда** (опционально, если усиливаем E-E-A-T)
12. **Сквозной футер**

**Доп. блоки на усмотрение проектировщика:** преимущества проекта, порядок оплаты, слайдер «другие проекты», галерея, видео-облёт.

## Страница услуги девелопера

(Применяется для каждой из услуг: строительство / управление / сопровождение / юр / архитектура / продажа земли / агентство.)

1. Промо-баннер с заголовком услуги + CTA
2. Что входит в услугу (3-6 пунктов)
3. Кейсы по этой услуге (карточки)
4. Процесс работы по услуге (таймлайн)
5. Команда, отвечающая за услугу
6. FAQ по услуге
7. Цена / диапазон / «по запросу»
8. Форма обратной связи

## Страница локации

(Применяется для каждой локации: Чангу / Семиньяк / Убуд / Улувату / Табанан / …)

1. Промо-баннер + описание локации
2. Преимущества локации (инфраструктура, ROI-потенциал, климат, языки)
3. Карта локации с pin'ами своих объектов
4. Листинг объектов в этой локации
5. Гайд «жизнь в локации» (информационный контент для SEO)
6. Блог-статьи про эту локацию (перелинковка)
7. Форма «подобрать объект в этой локации»

## Страница инвест-инструментов

### Калькулятор рентабельности
- Поля: тип объекта, локация, бюджет, период владения, режим (аренда / перепродажа)
- Результат: примерная ROI / cashflow / break-even
- CTA: «получить детальный расчёт от менеджера»
- Disclaimer: «расчёт примерный, фактическая доходность может отличаться»

### «Инвестиции на Бали» / «Diversify portfolio» (хаб-страница)
- Объяснение инвест-логики
- Сравнение с другими активами
- Юр-нюансы (freehold / leasehold для иностранцев)
- Калькулятор-виджет (короткая версия)
- CTA на консультацию

## Страница блог-статьи (YMYL-обязательная разметка)

1. Сквозной хедер
2. **Заголовок (`<H1>`)**
3. **Оглавление** (якорные ссылки на `<H2>`)
4. **Изображения** с `alt` (ключевыми словами)
5. **Текст** с чёткой иерархией `<H1>`/`<H2>`/`<H3>` + анкорные ссылки
6. **Блок об авторе** — фото + ФИО + должность + соцсети (опц. ссылки на цитируемые статьи)
7. **Перелинковка с другими статьями**
8. **Форма обратной связи**
9. Сквозной футер

**Обязательно:**
- schema.org/Article — на всех элементах
- alt на всех изображениях
- автор — реальный человек с верифицируемыми контактами

## Страница «О компании» (RE-режим)

Доп. к общему чеклисту:
- Лицензии (с фотокопиями) — критично для YMYL
- Команда с фото + ФИО + LinkedIn
- История компании с датами
- Награды и сертификации
- Партнёрства с банками / нотариусами / страховыми
- Финансовая стабильность (если можно показывать оборот / кол-во проектов)

## Страница корпоративного партнёрства (для агентов / юристов / архитекторов)

Аналог B2B-страницы «Дилерам» в UPH-кейсе:
- Промо-баннер
- Кому подходит партнёрство
- Что получает партнёр (комиссия / co-branding / лиды)
- Процесс присоединения
- Форма заявки (с полями вид деятельности, опыт, регион)
- (НЕ показывать существующих партнёров на карте — демотивирует новых, как в UPH)

## Специальные состояния (RE)

- **Пустой результат поиска объектов** — форма «оставьте параметры, подберём» вместо тупика
- **Объект продан / снят** — карточка остаётся в индексе с пометкой «sold» + CTA «посмотреть похожие в этой локации» (для SEO-перелинковки)
- **Калькулятор без всех параметров** — показать «примерную» ROI с дисклеймером + CTA «получить точный расчёт»

## Дополнительные паттерны блоков из созвонов Allex Villas

На основе разбора 6 созвонов (см. `case-allex-villas/process-from-calls.md`). Это паттерны, которые всплыли в живом диалоге и не покрыты выше.

### Интерактивная карта проектов на главной

Не Google/Yandex карта, а **статичная стилизованная карта региона** с поинтами проектов. При наведении/клике на поинт - попап-карточка с фото + краткое описание + кнопка «подробнее» -> страница проекта. Решает задачу «показать масштаб без слов "у нас много"».

Анти-паттерн: «просто полуостров с точками без названия районов» - конкуренты делают именно так, и это не работает.

### Пара «рендер vs реализация» через бегунок

На странице завершённого проекта - блок где бегунком переключается рендер -> фотореализация. Особенно сильно работает, когда «они совпадают, но всё равно видно, где реальная фотка».

### Карточка проекта на листинге

Минимальный набор полей (с созвонов 02-04):
- Название комплекса
- Одна фотография
- Локация (район)
- Состав юнитов (виллы / апартаменты / номера) - текстом
- Сколько юнитов всего, сколько свободно (для завершённых проектов)
- ROI (только если «вкусная цифра», иначе скрываем)
- Цена «от» (только для проектов с привлекательной стартовой ценой; на дорогих скрываем)
- Статус (строится / сдан / закрыт; для строящихся - этап «N из M»)
- Кнопка «Узнать больше» -> страница проекта
- (Опц.) пиктограмма «продаётся / продано»

**Маркетинговый приём «осталось N юнитов»** - используется, но не везде, иначе перебарщиваем (теряет силу).

### Якорное меню по странице проекта

Длинная страница проекта закрывается якорным меню (закреплённым при скролле). Минимальный набор якорей: **Локация / Инфраструктура / Юниты / Доходность**. Это «4 кита проекта» в RE-нише.

### Блок «Прогноз доходности» - 2 варианта

- **Вариант А (в рамках базового бюджета)**: PDF-«шахматка» с просчётами + кнопка скачать.
- **Вариант B (за доп. бюджет)**: интерактивный блок:
  - Анализ рынка (как мы получаем цифры; что у нас за команда экспертов)
  - Прогноз заполняемости в динамике (диаграмма)
  - Прогноз роста туристического потока (диаграмма)
  - Прогноз роста стоимости аренды (диаграмма)
  - Финансовая модель по юниту (период / этап строительства / прогноз дохода)
  - CTA «получить детальный расчёт»

**Анти-паттерн «гарантированная доходность»**. Если в договоре ROI не закреплён (а юридически это редко) - убирать слово «гарантированная» полностью. Заменять на «ожидаемая», «прогнозируемая», «исторический пример».

### Блок «Кейсы доходности» (для завершённых проектов)

Альтернатива прогнозам - реальные кейсы:
- Этап заключения сделки (внутренняя отделка / на старте)
- Дата сделки
- Тип юнита
- Прогнозируемая доходность (на момент покупки)
- Фактическая доходность
- Срок окупаемости
- (Опц.) впечатления клиента (без имён, если нельзя)

### Блок «Финансовая модель» как лидмагнит

В Allex Villas взяли вариант: **не показывать финмодель целиком на сайте**, а упоминать что «есть аналитики, есть модель» + CTA «оставьте заявку, расскажем». Цель - переводить интерес в консультацию.

### Интерактивная схема комплекса / юнита

- На странице проекта: **схема комплекса сверху** + поинты на корпусах/бассейнах/спа -> при наведении карточка с описанием.
- На карточке юнита: **схема юнита с интерактивом** - при наведении на мебель «производство компании», на кондиционер «управляется системой умный дом», на материалы «своё производство». Цель - ещё раз транслировать УТП «полный цикл» не текстом, а через интерактив.

### Блок «Как мы строим наши проекты» на главной

Слайдер по этапам: проектирование -> производство материалов -> строительство -> отделка -> мебель -> управление. Картинки нарастают, доводя до итогового вида проекта. Решает SEO-задачу «текстовый объём + ключевые запросы про процесс» и продуктовую «убрать страх "вы только продаёте"».

### Блок «Нас рекомендуют»

Логотипы СМИ + партнёров. Анти-паттерн: «универсальный набор для всех языковых версий». Логотипы должны быть локальные: для RU - РБК, Коммерсантъ; для EN - Forbes, CNN. Иначе аудитория не считывает.

### Блок «Пара "вы зашли в проект - вот другие"»

В Allex Villas - слайдер других проектов внизу карточки + перелинковка из «Локации». Не «слайдер из миллиона других проектов кричаще внизу», а элегантно: 3-5 «других в этой локации» / «других флагманов».

### Блок «Запишитесь на консультацию» - расшифровка

Анти-паттерн: «бесплатная консультация» - инвесторы воспринимают как впаривание. Решение: явно расписать «на консультации вы узнаете: X, Y, Z» (3-5 пунктов от отдела продаж). Например: «почему именно эти проекты инвестиционно привлекательны / на что обращать внимание при выборе застройщика / какие локации сейчас востребованы / стратегии инвестирования». Конкретика снимает страх «впарят».

### Поле «удобное время для звонка» в формах

Если ЦА международная - часовые пояса разные. Не делать автоопределение (часто глючит), а просто поле «удобное время» + определять часовой пояс по коду номера телефона на бэкенде.

### Страница «Агентам» как отдельная страница (а не блок на главной)

Структура (с Allex Villas):
- Промо «Зарабатывай с нами»
- Почему Бали (или другой регион) - для агентов из других рынков
- Условия для агентов (комиссии, эксклюзивы, бронирование)
- Поддержка и сопровождение 24/7 (закрытые чаты, материалы)
- Кейсы партнёров (без имён, в анонимной форме)
- Отзывы брокеров (видео если есть)
- Обучение и мероприятия
- Ссылки на закрытые соцсети для агентов
- Форма «Стать партнёром»

**Анти-паттерн «личный кабинет агента»**: на Allex Villas пробовали - не сработало. Решение - не строить, ссылаться на закрытые чаты / Telegram-группы.

**Контактные потоки**: разделить телефоны/email для инвесторов и агентов в подвале сайта - даже если на старте оба ведут на тех же менеджеров. Это даёт возможность позже разделить потоки без перевёрстки.

### Блок «Карьера» / «Вакансии»

Не публиковать список вакансий (постоянно меняется), а сделать **блок «Хочешь работать с нами?»** с формой загрузки резюме. Размещать на странице «О нас», не выделять в отдельный пункт меню.

### Подвал и «гибкие условия оплаты»

Блок «Порядок оплаты» на странице проекта пишется обтекаемо: «гибкие условия оплаты, подробности у менеджера». Причина - реальные условия часто отличаются от официальных (рассрочка, крипта и т.п.), и фиксировать конкретику публично нельзя.

### Соцсети в подвале - какие показывать

Только активные с реальной аудиторией для языкового рынка сайта. На Allex Villas: не стали добавлять Telegram, потому что «он у нас русскоязычный, а сайт английский». Не пихать всё подряд - смотреть, что соответствует языку версии.

## Multi-language

- EN основной для международного рынка (Бали, Дубай, Турция, Тайланд)
- RU обязателен для русскоязычных рынков (Сочи, Краснодар, Алматы)
- Языковые версии — отдельные папки (`/en/`, `/ru/`) или поддомены, **не cookie-switch**
- Hreflang-разметка обязательна
- Перевод не машинный — носителем языка, особенно для YMYL-блогов

## Технические требования (RE)

- **Schema.org:**
  - `Organization` в футере
  - `RealEstateListing` или `Product` на карточках объектов
  - `Article` на блог-статьях
  - `BreadcrumbList` на всех страницах
  - `FAQPage` на FAQ-блоках
  - `LocalBusiness` для региональных страниц
- **Mobile-first** — >60% трафика недвижимости мобильный
- **Core Web Vitals:** цель ≥60/70 (mobile/desktop) — большинство конкурентов слабее
- **Lazy loading** — на галереях и картах
- **Edge CDN** — обязательно для multi-region трафика
