# Чеклист вопросов к клиенту

Список вопросов, которые обязательно задать клиенту на каждом этапе проектирования сайта. Из кейсов [UPH](case-uph/) (B2B-каталог) и [Allex Villas](case-allex-villas/) (B2C-недвижимость) + обобщение.

## Какой режим вопросов выбрать
- **B2B-каталог** (UPH-режим) — производитель/дистрибьютор оборудования, ПО, услуг для бизнеса. Каталог + госзакупки + дилеры + сотрудничество.
- **B2C-недвижимость** (Allex Villas-режим) — застройщик / девелопер / агентство. Листинг проектов + продажа + аренда + услуги застройщика. См. отдельный блок ниже.

Дальше по умолчанию — общий B2B-каталог-чеклист. Спецблок «Недвижимость» — в конце документа.

## На этапе паспорта

**Бизнес-цели:**
- Какая главная задача сайта? (лидогенерация / узнаваемость / удержание / расширение)
- Есть ли KPI (лиды в месяц, конверсия, SEO-позиции)?
- Какие дополнительные цели - лояльность, имидж, информирование?

**ЦА и конкуренты:**
- Кто основные клиенты? Сегменты (госсектор, B2B-производство, частный бизнес)?
- Кто конкуренты? Какие у них сайты?
- Что в них нравится / не нравится?

**Продукт:**
- Как устроен каталог в 1С / ЦРМ? Какая группировка (по технологии / отрасли / виду)?
- Сколько позиций, категорий, под-категорий?
- Что из каталога публично, что коммерческая тайна?
- Есть ли экспортная активность (нужны языковые версии)?

**Команда и процессы:**
- Кто будет следить за актуальностью контента (price, новости, каталог)?
- Есть ли в ОП разделение менеджеров по отраслям?
- Как работают с входящими лидами (какие поля нужны в форме)?
- С какими мессенджерами работают (Max, WhatsApp, Telegram)?

## На этапе структуры каталога

- Какой вариант группировки каталога удобнее (3 варианта на выбор)?
- Как это коррелирует с 1С / ЦРМ?
- Нужны ли фильтры в каталоге? По каким параметрам?
- Есть ли модификации одного оборудования? Как их показывать?

## Госзакупки (B2G)

- Работаете ли по 44-ФЗ / 223-ФЗ?
- Есть ли отдельные преимущества для госзаказчиков, которые стоит подчеркнуть?
- Какие документы/сертификаты нужны для госзакупок?

## Дилеры

- Какой процесс входящей заявки на дилерство? Анкета или обычная форма?
- Какая информация о потенциальном дилере нужна в первом контакте?
- Показываем ли текущих дилеров? (**По умолчанию - нет**, демотивирует)
- Есть ли презентация для дилеров?
- Есть ли сервисные центры, которые имеет смысл показать на карте?

## Сотрудничество (КБ, вузы)

- Какие типы партнёров интересны (КБ, вузы, другие)?
- Характеристики "идеального" партнёра: вид деятельности, опыт, документы (чтобы фильтровать мусорные лиды)?
- Какие варианты сотрудничества возможны (внедрение, финансирование, производственная база)?
- Можно ли показывать с кем уже работаете? Какие проекты приемлемо раскрыть?
- Какие вузы уже сотрудничают?

## Proof и портфолио

- Есть ли благодарственные письма со сканами? Можно ли их публиковать?
- Логотипы клиентов - каких можно показать?
- Портфолио объектов - фото зданий с оборудованием, отзывы представителей - что приемлемо?
- Сертификаты - какие важно подсветить?

## Федеральные законы и регулирование

- Есть ли единый закон, который обязывает устанавливать ваше оборудование?
- Или каждая отрасль регулируется отдельно (часто именно так)?
- Если по отраслям - раскрывать закон на странице отрасли, не на странице под-категории товара.

## Сервис

- Сколько гарантийный срок?
- Есть ли послегарантийное обслуживание? (отдельный блок для удержания)
- Можно ли обучать сотрудников клиента?
- Есть ли расходники, замена, апгрейд?

## Контент

- Есть ли готовые тексты преимуществ, описаний?
- Есть ли фотобанк оборудования высокого качества?
- Есть ли видео работы оборудования?
- Готовы ли писать статьи в блог (для SEO + информационной миссии)?

## Технические и CMS

- Предпочитаемая CMS (Tilda / Bitrix / WordPress / кастом)?
- Интеграции с 1С, CRM (amoCRM, Bitrix24)?
- Какая аналитика (Метрика, GA)?
- Какие мессенджеры интегрируем?
- Нужна ли админка для контент-менеджера?

## На каждом созвоне - обязательные практики

1. **Фиксировать "на обсуждение"** - если вопрос не закрыт, пометить прямо в Miro.
2. **Показывать примеры конкурентов** - клиент лучше реагирует на "как у этих" чем на абстракции.
3. **Не проходить всё за один созвон** - дать клиенту время почитать самостоятельно после демо.
4. **Следующий шаг на каждый блок** - что нужно от клиента (текст, фото, контакты ответственного).

# Спецблок: Недвижимость (B2C-девелопер / агентство)

Используется для застройщиков, агентств недвижимости, инвест-платформ. На основе кейса Allex Villas (Бали).

## На этапе паспорта (доп. к общим вопросам)

**Бизнес-модель:**
- Полный цикл (строим + управляем + продаём) или одна функция?
- Какие услуги в портфеле: строительство / управляющая / сопровождение / юр / архитектура / продажа земли / агентство?
- Свой портфель объектов или брокеридж?
- Сегмент: эконом / средний / премиум / luxury / инвест?

**Объекты:**
- Какие типы объектов: вилла / квартира / земля / отель / коммерция?
- Какие типы сделок: аренда / freehold / leasehold / hotel-rental-pool?
- Какие локации работаете (для региональных кластеров SEO)?
- Сколько объектов в портфеле сейчас, сколько планируется?

**Инвест-инструменты:**
- Есть ли расчёт ROI / прогноз доходности по проектам? Как презентуете?
- Нужен ли калькулятор рентабельности на сайте?
- Какая средняя ROI у проектов (для title/description)?

**ЦА:**
- Кто основной покупатель: иностранный инвестор / локальный покупатель / B2B-партнёр?
- На каком языке думает покупатель (нужны какие языковые версии)?
- Какие страны-источники трафика (для гео-рекламы и языков)?

**Юр-нюансы:**
- Какая структура владения для иностранцев (freehold через юр.лицо, leasehold, …)?
- Какие лицензии у компании (важно для E-E-A-T)?
- Регуляторные требования (как с 44-ФЗ для России)?

## На этапе структуры (для недвижимости)

- Все 5 осей фильтрации (тип / собственность / бюджет / локация / спальни) актуальны? Какие выкидываем?
- Разделять ли «Разработки» (что строим) и «Продажа» (что продаём)?
- Как разносить аренду от продажи (отдельный раздел или фильтр)?
- Сколько типов листингов (готовые / строящиеся / luxury / discount)? Если ≥2 — нужен **уникальный FAQ** на каждом, иначе каннибализация.
- Нужна ли «Управляющая компания» в основных услугах? Если да — отдельный микро-сайт внутри.

## SEO-этап (специфика недвижимости, YMYL)

- Есть ли в команде штатный/внешний SEO-специалист, или мы подключаем своего?
- Готовы ли вкладываться в блог с авторами (фото+ФИО+контакты)? Это критично для YMYL.
- Согласны ли на schema.org-разметку Article/RealEstateListing/Organization (это работа для разработчика)?
- Есть ли отзывы клиентов, которые можно публиковать с именами?
- Есть ли упоминания в СМИ, ссылки от авторитетных источников?
- Сколько готовы делать статей в блог в месяц (для индексации информационных кластеров)?

## Дизайн-этап (мудборд → концепты)

- Есть ли референсы (сайты, которые нравятся визуально)?
- Какая «эмоция» бренда: премиум / технологичный / семейный / активный / экзотика?
- Есть ли фирменные цвета, шрифты, логотип, гайдлайн?
- Есть ли фото-банк объектов (свои фото, рендеры, видео-облёты)?
- Сколько итераций концептов готовы пройти (обычно 2–3)?

## Прогноз доходности

- На какую цифру ROI можно ссылаться публично (с обоснованием)?
- Есть ли калькулятор/формула, по которой считаем?
- Какие переменные пользователь должен ввести (площадь, локация, период владения)?
- Хотим показывать «оптимистичный/реалистичный/пессимистичный» сценарии?

## Команда сайта (E-E-A-T)

- Можем показать команду (фото, ФИО, должности, опыт)?
- У ключевых сотрудников есть LinkedIn / соцсети для ссылок?
- Кто может выступать автором блог-статей (ставить ФИО, фото, экспертность)?
- Есть ли партнёрские агенты, юристы, архитекторы — выделить как отдельных авторов?

## Multi-language

- Какие языки нужны на старте (EN / RU / ZH / FR / …)?
- Есть ли носитель/переводчик для каждого языка?
- Делаем дублирующую структуру или маршрутизацию по поддоменам/папкам (`/en/`, `/ru/`)?
- Hreflang-разметка нужна?

## CRM и трекинг

- Какая CRM (amoCRM, HubSpot, Bitrix24, Pipedrive, кастом)?
- Есть ли разделение менеджеров по локациям/типам объектов/языкам?
- Какие конверсии трекаем (заявка / звонок / Telegram / WhatsApp / скачивание PDF / калькулятор)?

## Реальные вопросы с созвонов Allex Villas

Это вопросы, которые на самом деле прозвучали на 6 созвонах проекта Allex Villas (см. `case-allex-villas/process-from-calls.md`). Используйте как чеклист в RE-проектах того же класса.

### С созвона 01 (паспорт + конкуренты)

- Какие фишки конкурентов мы берём в backlog для структуры? (по каждой: «зачем нашей ЦА» -> берём / откладываем / отказываемся)
- Языковая стратегия релиза: один язык на старте или сразу два? (созвон 01: «не вижу смысла выпускать релиз только на английском или только на русском»)
- На каком языке мышления проектируем (русский / английский / другой)? Кто переводит - носитель или машина?
- Есть ли у компании политика «не показывать команду / офис»? Когда планируете её менять?
- Есть ли видео-трансляции стройки, и если да - на всех ли проектах их можно показывать (учитывая срывы по срокам)?
- Какой хостинг сайт сейчас крутится? Есть ли проблемы с трансграничными кабелями (для международных проектов)?
- Можно ли публиковать награды / упоминания в СМИ / правительственные взаимодействия?
- Есть ли политика по отзывам - текст с фото / видео / на внешних платформах (Airbnb, TripAdvisor)?

### С созвона 02 (демо структуры)

- Сколько проектов в портфеле сейчас? Сколько в продаже? (для решения по фильтрам и слайдерам)
- Все 5 осей фильтрации листинга актуальны? Окупаемость лучше показывать или скрывать? (Allex Villas решили скрыть, потому что «у нас 5-7 лет везде одинаково»)
- Есть ли отдельные контакты для агентов и инвесторов или единый телефон? (предположение - меняется по политике клиента)
- Есть ли какие-то «секретные» блоки, которые показываются избранным/закрытым каналом (агенты с личным кабинетом)?
- Можно ли называть имена клиентов в кейсах? Можно ли публиковать их фото?
- Какие ресурсы социальных сетей вести с английским контентом? (рассинхрон языковых аудиторий)
- Можно ли показывать рейтинг с внешних платформ (Airbnb 5.0 + N тысяч отзывов) - и где?
- Чем в реальности занимается ваша управляющая компания? (по пунктам, для блока «УК делает X / Y / Z»)
- Гарантированная доходность - правда ли закреплено в договоре, или это маркетинговая формулировка? (если не закреплено - убирать слово «гарантированная»)
- Есть ли у вас портреты целевых персон? (кейс Allex Villas: ОП дал готовый документ, проектировщик переделал его в сегментацию)

### С созвона 03 (бриф на дизайн)

- Какую основную эмоцию должен передавать дизайн? (Allex Villas: «не кричащий люкс, а спокойная уверенность, тихая роскошь, slow living»)
- Что мы НЕ хотим? (анти-эмоция; Allex Villas: «кричащий люкс» в анти-список)
- Есть ли цвета, которые вы исключаете из бренд-гайдлайна (даже если они в нём прописаны)? (Allex Villas: жёлтый исключили)
- Какое допустимое скругление элементов? (Allex Villas: «не более 5%, в большинстве случаев избегать»)
- На что больший акцент - смыслы или фото? Какой объём качественного фотобанка у вас сейчас?
- Согласны ли использовать ChatGPT / нейроконтент? (Allex Villas: «никаких эмодзи - в прошлый раз дизайнер тексты с ChatGPT забыл подчистить»)
- Используете ли вы видео на фоне главной? (Allex Villas: «нет, тяжёлое, тормозит, точно не нужно»)
- Сколько итераций концептов готовы пройти? (Allex Villas: «я объяснила, что норма - 2-3 итерации с комбинированием»)
- Есть ли срочные параллельные посадки (под рекламу) - которые можно делать на основе нашего прототипа их штатным дизом, не подмешивая в основной процесс?

### С созвона 04 (мудборд)

- Когда листаете референсы - что вызывает «сразу хочется закрыть»? Что наоборот «прям нравится»?
- Что нравится конкретно: расположение / шрифт / соотношение картинок / эффект анимации?
- Закреплённое боковое меню на странице проекта - окей или мешает?
- Преlook / pre-loader - используем как способ донести ключевой message, или отказываемся?
- Можно ли использовать монограмму (логотип) на каждом текстовом блоке? (Allex Villas: «нет, прям на каждый блок не стоит»)
- Поиск на сайте - нужен или нет? (Allex Villas: «не понимаю функционального назначения, давай SEO-эксперт примеры пришлёт»)
- Слайдер «листается вниз» (а не вбок) - окей или непривычно?

### С созвона 05 (защита первой итерации концептов)

- Чёрный или белый концепт ближе по эмоции? (Allex Villas: «чёрный нравится больше всего, но из белого хочется взять блоки X, Y»)
- Какие блоки точно уйдут в «комбинированный финальный концепт»?
- Когда вы будете давать детальный фидбэк - с экрана сейчас или пройдёмся с визуальными правками в чате?
- Айпад / телефон - вы будете адаптив сами проверять до согласования?

### С созвона 06 (защита финального концепта)

- В чём конкретно «не премиум»? (Allex Villas: «жирный шрифт, гигантизм, нет иерархии»)
- Можно ли заменить жирный шрифт цветом для акцентов? (универсальный приём для строгих RE-проектов)
- Дизайнер может сам перекрашивать фотоконтент в ЧБ / убирать фон / адаптировать под композицию? (вопрос границ ответственности)
- Если вы не можете описать словами «нравится / не нравится» - можно скинуть Pinterest-референсы прямо в чат?

### Контент-блокер - сквозной вопрос всех созвонов

- Кто на стороне клиента отвечает за наполнение контента (тексты, фото, награды, отзывы, FAQ)?
- Когда мы получим реальные данные (даже черновые)? Без них дизайн поедет на этапе верстки.
- Будут ли у блог-статей реальные авторы (с фото, ФИО, ссылками на соцсети) или только «admin»?
- Сколько фактически наград / отзывов / логотипов «нас рекомендуют» вы можете предоставить? (от этого зависит, рисуем ли слайдер или статичную плитку)

## Что НЕ показывать (анти-паттерны для недвижимости)

- ❌ **Точные адреса объектов до сделки** — security-риск.
- ❌ **Цены без обоснования / устаревшие** — Google и инвестор накажут.
- ❌ **Слайдер из ОТКЛЮЧЁННЫХ отзывов** — если копили цитаты «вручную», сразу указать ФИО или дату; без этого выглядит фейково.
- ❌ **«Гарантированная доходность 30%»** — некорректно с т.з. регулятора. ROI как «ожидаемая» / «исторический пример».
- ❌ **«Ничего не найдено» в поиске объектов** — форма «оставьте заявку, подберём» вместо тупика.
